# Learning curves (from train_log.json)

Validation learning curves (per epoch) for the FW_Event_Net experiments, generated by
`eventnet/plot_curves.py`. **Curves are VALIDATION** (the 7 train scenes' held-out frames),
which is what model-selection used; the headline TEST numbers (3 held-out scenes) are the
single best-checkpoint values in `EXPERIMENT_LOG.md` / `RESULTS.md`. Note the val↔test gap
(val ~0.70 vs test ~0.55) is the cross-scene domain gap discussed in those docs.

| file | what it shows |
|---|---|
| `v1_feature_ablation.png` | V1 (K=4) t_only/t_dt/ta/taw/tdta/tdtaw — val F1 / glass / ghost |
| `v1_ksweep.png` | V1 tdtaw K=1/2/4/8 — ghost collapses at K=1, K=4 best |
| `v2_feature_ablation.png` | V2 (K=4, seed42) ta/taw/tdta/tdtaw |
| `arch_compare_taw.png` | taw across V1 / V2 / V2-base-fair / v2sa-fair — V1 noisier & lower, V2 smoother/higher |
| `v2sa_fair_retest.png` | fair-recipe base vs v2sa (2 seeds): val F1 / ghost / train loss — base ≈ v2sa (neutral) |
| `negative_experiments.png` | taw ctrl vs +decomp / +behind_E / +NeRF-T / V-REx / glass-weight — val F1 / glass |

Regenerate: `PYTHONPATH=<repo>/src uv run python -m eventnet.plot_curves`.
